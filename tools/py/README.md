# Python Job Hunt Utilities & CV Tailoring Suite

A comprehensive Python-based ecosystem designed to automate, optimize, and benchmark the job application lifecycle. This workspace houses tools for scraping job listings, evaluating role fit with local LLMs, tailoring CVs for ATS platforms, validating layout syntax, and evaluating outputs using multiple AI benchmark frameworks.

---

## Workspace Structure

The project is organized into modular components, each handling a specific stage of the job search and CV customization process:

```bash
tools/py/
├── cv/                      # Core CV templates, examples, and local compiler scripts
│   ├── example/             # Markdown examples (header.md, body.md, footer.md)
│   ├── prompts/             # ATS-optimized system and user templates
│   └── tools/               # Local generators (PDF conversion, prompt builders, process tools)
├── ui_composer/               # Streamlit web dashboard for interactive tailoring and previews
├── job_finder/              # Automated job scraper (Indeed, LinkedIn, StepStone) & LLM matcher
├── md_tools/                # Pydantic schemas representing CV components & CLI syntax checker
├── tailor_cv_eval/          # Benchmarking suites (DeepEval, Promptfoo, LlamaIndex, Ragas, etc.)
└── helpers/                 # Shared LLM clients (Ollama, Gemini) and global Pydantic definitions
```

---

## Component Details

### 1. 📄 CV Tailoring Tools (`cv/`)

Contains templates, prompts, and local compilation tools that handle the data flow for CV rendering:

- **`tools/process_cv.py`**: Performs automatic cleanup, markdown fixing, and syntax corrections.
- **`tools/md2pdf.py`**: Compiles markdown files into highly professional PDF resumes using `pandoc` and `pdflatex` (with LaTeX alignment).
- **`prompts/`**: Employs system instructions focused on ATS keywords, keyword density, and zero-hallucination rules.

### 2. 💻 Tailor UI Dashboard (`ui_composer/`)

An interactive, browser-based Streamlit application to visualize and coordinate CV tailoring:

- **Hydrated Prompt Generator**: Intersections master CV data with job descriptions to build LLM prompts.
- **Side-by-side Editor**: Offers markdown editing tabs alongside live PDF preview displays.
- **Formatting Tools**: Features **Check CV** (syntax checking) and **Fix** buttons right inside the composition workspace to clean up markdown layouts.

### 3. 🔍 Automated Job Finder (`job_finder/`)

A fully automated crawler and role suitability reviewer:

- **Scraper**: Collects new postings from **Indeed**, **LinkedIn**, and **StepStone** (configurable in `scraper/`).
- **Reviewer**: Passes JDs to local LLMs (configured via Pydantic matching criteria) to evaluate compatibility, filtering out non-matching listings automatically.
- **CLI Commands**: Supports checking single files (`check_jd.py`) or full crawl-and-evaluate cycles (`fetch_and_match.py`).

### 4. 🗂️ Markdown Parser & Validator (`md_tools/`)

Defines strict Pydantic model representations for resume sections (Header, Body, WorkExperience, PersonalProjects, Courses, Degree, Languages, Summary, and Skills) to ensure structural integrity:

- **Bidirectional Conversions**: All models support `to_string()` (serializing to clean Markdown) and `from_string()` (deserializing back to Pydantic objects).
- **CLI Validator**: Running `cli.py` prints Pydantic validation errors and structural mismatch alerts with detailed line numbers.

### 5. 📊 Benchmark Evaluation Suite (`tailor_cv_eval/`)

Measures and compares CV variants generated by local/online LLMs against standard ground truths:

- **Engines Supported**:
  1. **DeepEval**: Pytest-based G-Eval correctness check.
  2. **Promptfoo**: Interactive dashboard scoring semantic similarity, LLM rubrics, and ROUGE-N scores.
  3. **LlamaIndex**: Native correctness and relevance evaluation.
  4. **Ragas**: Faithfulness and answer relevance checks.
  5. **TruLens**: Logs generation metrics and score patterns into a local DB.
  6. **Giskard**: Checks factual alignment.
  7. **Text-Grad**: Stanford backpropagation-through-text to generate gradient feedback.

## Getting Started

### Prerequisites

- **Python**: Version `3.12` or higher (configured in `.python-version`).
- **Pandoc & LaTeX**: Required for PDF rendering:

  ```bash
  sudo apt-get install pandoc texlive-latex-extra texlive-xetex
  ```

- **Ollama**: For local models evaluation and inference:

  ```bash
  ollama pull gemma4:e2b
  ollama pull llama3.1:8b
  ```

### Installation

We use `uv` to manage dependencies. Run the following command in this directory:

```bash
uv sync
```

## Quick Commands Reference

### Running the UI Dashboard

Start the Streamlit interface locally:

```bash
uv run streamlit run ui_composer/app.py
```

### Job Search & Matching CLI

Scrape and filter job listings based on LLM suitability checks:

```bash
uv run python job_finder/cli/fetch_and_match.py
```

### Checking CV Markdown Syntax

Run the strict parser over a local markdown resume to check for ATS parsing errors:

```bash
uv run python md_tools/cli.py cv/example/body.md
```

### Running Tests and Linters

Validate code formatting and model roundtrip assertions:

```bash
# Run tests
uv run pytest md_tools/models_test.py md_tools/parse_test.py

# Lint codebase
uv run ruff check .
```
